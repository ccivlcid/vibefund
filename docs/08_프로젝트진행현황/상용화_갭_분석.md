# VibeFund 상용화 갭 분석 (PRD v2.0 기준)

PRD v2.0·기능 요구사항 대비 **현재 구현 상태**와 **상용화 전 보완 필요 항목**을 정리한 문서입니다.

---

## 1. 요약 (PRD v2.0 MVP 기준)

| 구분 | PRD v2.0 요구사항 | 현재 구현 | 갭 |
|------|-------------------|-----------|-----|
| **인증** | 이메일 + OAuth(Google) | 이메일만 (JWT·쿠키) | OAuth 미구현 |
| **프로젝트** | 제목, 한줄설명, URL, **카테고리**, **상태**, 썸네일 | 카테고리·상태 UI 없음 | UI 보강 |
| **서비스 체험** | iframe 기반 | ✅ 구현됨 | — |
| **검증 질문** | 고정 3문항 응답 저장·노출 | ❌ 없음 | **전체 미구현** |
| **댓글** | 작성·**대댓글**·**신고** | 작성만 | 대댓글·신고 미구현 |
| **Dev Log** | 업데이트·버전 기록 | 업데이트 탭 있음 | 버전/이력 강화 선택 |
| **Vibe Score 1.0** | 가중치 기반 점수 | ❌ 없음 | **전체 미구현** |
| **랭킹** | 인기·상승·댓글·Vibe Score | ❌ 없음 | **전체 미구현** |
| **관리자** | 검수·**댓글 신고 처리**·사용자 차단 | 검수 있음, 신고 플로우 없음 | F-054 미구현 |
| **마이페이지** | 프로필·내 프로젝트·활동 | ✅ 구현됨 | — |
| **법적·운영** | — | 이용약관·개인정보처리방침 없음 | 필수 페이지 없음 |
| **비기능** | 로딩 3초, 반응형, 가용성 99% | 반응형 있음, 성능·모니터링 미측정 | 측정·개선 필요 |

**참고**: 기존 **펀딩(목표·진행률·후원)** 기능은 코드에 존재하나, PRD v2.0 Phase 1 MVP에는 **제외**. 유지할지 Phase 2로 미루는지는 정책 결정.

**결론**: v2.0 MVP 기준으로 **검증 질문·Vibe Score·랭킹·대댓글·신고** 등이 갭이며, 상용 서비스 전 위 항목 보강 + 법적 페이지가 필요합니다.

## 2. 기능별 상세 갭

### 2.1 사용자 관리 (F-001 ~ F-003)

| 요구사항 | 구현 여부 | 비고 |
|----------|-----------|------|
| F-001 이메일 회원가입 | ✅ | Supabase + bcrypt, 중복 검사 |
| F-001 OAuth(Google) | ❌ | API 명세에만 있고 구현 없음 |
| F-002 로그인/로그아웃 | ✅ | JWT + httpOnly 쿠키 |
| F-003 프로필 관리 | ✅ | `PATCH /users/me` |

---

### 2.2 프로젝트 허브 (F-010, F-011)

| 요구사항 | 구현 여부 | 비고 |
|----------|-----------|------|
| F-010 제목, 한줄설명, URL, 썸네일 | ✅ | `/projects/new` |
| F-010 **카테고리** (AI/SaaS/Automation/No-Code) | ⚠️ | API optional, **UI 없음** |
| F-010 **상태** (Prototype/Beta/Live) | ⚠️ | API 기본값만, **UI 선택 없음** |
| F-011 iframe 서비스 체험 | ✅ | 상세(view)에서 iframe |
| F-011 **검증 질문 3문항** | ❌ | **미구현** — (1) 사용 의향 (2) 월 지불 금액 (3) 개선점 |
| F-011 **대댓글** | ❌ | 스키마 parent_id 있으나 **UI 없음** |
| F-011 Dev Log 탭 | ✅ | 업데이트 탭 존재 (버전/이력 강화는 선택) |

**권장**: 프로젝트 폼에 카테고리·상태 추가. **검증 질문** — 응답 테이블·API·상세 페이지 폼 추가.

## 5. 상용화 전 우선순위 제안 (v2.0 MVP 기준)

### P0 (런칭 전 필수)

1. **이용약관·개인정보처리방침** 페이지 및 푸터·가입 화면 링크
2. **프로젝트 생성 폼**: 카테고리(AI/SaaS/Automation/No-Code), 상태(Prototype/Beta/Live) 필드 추가
3. **검증 질문 3문항**: 응답 스키마·API·상세 페이지 응답 폼
4. **댓글 대댓글**: parent_id UI·API 연동
5. **댓글 신고 + 관리자 처리**(F-054): 신고 API·관리자 «신고 접수» 목록·삭제/유지

### P1 (단기)

6. **Vibe Score 1.0**: 계산 요소·공식 정의, 저장/노출
7. **랭킹 페이지**: 인기·상승·댓글·Vibe Score 기준 목록
8. **OAuth(Google)** 로그인
9. **에러·가용성 모니터링** 도입

### P2 (중기)

10. OpenGraph 썸네일 자동 생성
11. 성능 측정 후 3초 목표 점검
12. 핵심 API 자동 테스트
