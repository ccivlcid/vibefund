# 하니스 엔지니어링 (Harnes Engineering)

AI 에이전트 환경 설계에 대한 개념, 실무 단계, 경쟁 우위를 정리한 참고 문서입니다.

---

## 1. 하니스 엔지니어링이란

- **정의**: 하니스 엔지니어링은 **컨텍스트 엔지니어링**, **MCP**, **스킬**, **에이전트 설정**을 포함한 전체 환경 설계입니다.
- **성능 영향**: 같은 AI 모델이라도 하니스에 따라 성능 차이가 큽니다.
- **업계 동향**: OpenAI도 AI 에이전트 환경 구축에 집중하고 있습니다.
- **향후 방향**: 컨텍스트 파일, **MCP** 연결, 스킬 정리를 시작해야 합니다. 이러한 흐름은 2026년을 '하니스의 해'로 만들 것이며, AI의 똑똑한 활용을 이끌어 낼 것입니다.

---

## 2. 시작을 위한 실질적인 조언

### 2.1 컨텍스트 파일 (`CLAUDE.md`) 작성

- 프로젝트 구조, 규칙, 패턴 등을 AI에 알려주는 하니스의 **기본 단계**입니다.
- 루트 또는 `.cursor/` 등에 `CLAUDE.md`(또는 프로젝트 규칙 파일)를 두고, AI가 참고할 수 있도록 정리합니다.

### 2.2 MCP 서버 설정

- AI 에이전트가 **외부 도구**를 사용할 수 있도록 연결을 설정하는 단계입니다.
- Cursor·Claude 등에서 MCP(Model Context Protocol) 서버를 연결하면 DB, API, 브라우저 등 도구를 에이전트가 호출할 수 있습니다.

### 2.3 스킬 파일 생성

- AI에게 **특정 작업에 대한 전문 지식**을 파일로 정리하여 제공하는 단계입니다.
- `.cursor/skills/` 등에 SKILL.md 형태로 "언제 사용할지, 어떤 절차를 따를지"를 정의하면 에이전트가 해당 작업 시 자동으로 참고합니다.

---

## 3. 경쟁 우위로서의 하니스

- 모델이 빠르게 발전하고 서로 **유사해지는** 경향이 있지만, **잘 구축된 하니스는 복제하기 어렵기 때문에** 미래의 경쟁 우위가 됩니다.
- 개인에게도 **AI 도구를 잘 설정하는 것**이 생산성과 품질 차이를 만듭니다.

### 4.1 에이전트 설정 정기 점검

주기적으로 다음을 확인합니다.

| 항목 | 확인 내용 |
