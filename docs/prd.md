# 1. 제품 개요 (Overview)

## 1.1 제품 정의

VibeFund는 노코드/바이브코딩으로 제작된 웹사이트 및 SaaS 프로토타입을 업로드하고,  
실제 사용자들이 체험한 뒤 후원(펀딩) 또는 얼리액세스 구매를 할 수 있는 플랫폼이다.

기존 크라우드펀딩과 달리,
- 단순 소개 페이지가 아니라
- **실제 서비스 체험 → 후원** 구조를 제공한다.

---

# 2. 문제 정의 (Problem Statement)

## 2.1 시장 문제

- 한국에는 노코드 창업자가 빠르게 증가 중
- 하지만:
  - 투자 받기 어려움
  - 사용자 검증 구조 없음
  - 기존 크라우드펀딩은 물리 상품 중심
  - Product Hunt는 펀딩 기능 없음

## 2.2 해결 방향

- 노코드 웹서비스 전용 펀딩 플랫폼 제공
- "아이디어 검증"을 목표로 설계
- 체험 기반 설득 구조 도입

---

# 3. 목표 (Goals)

## 3.1 MVP 목표 (Phase 1 - Korea)

- 회원가입/로그인
- 프로젝트 등록 (URL 기반)
- 웹사이트 미리보기(iframe)
- 펀딩 목표 설정
- **실시간 펀딩 진행률 표시** (결제는 MVP 1차 범위 제외, 추후 Stripe 도입)
- 댓글 기능
- **배포·운영**: Vercel Git 연동으로 빌드·배포 자동화

## 3.2 Phase 2 목표

- 멤버십(구독형 후원)
- 프로젝트 추천 알고리즘
- 제작자 프로필 페이지
- 성공 프로젝트 프로모션
- 데이터 분석 대시보드

# 11. 기술 스택 제안

리포지터리:
- **모노레포** (단일 리포에서 앱·공용 패키지 관리). pnpm workspace, Turborepo 권장. 상세는 docs/01_아키텍처/모노레포_구조.md 참고.

Frontend:
- Next.js
- Tailwind CSS

Backend:
- Next.js API Routes (모노레포 내 apps/web 또는 동일 앱)
- PostgreSQL (Supabase)
- Redis (선택, MVP에서 미사용 가능)

Infra:
- Vercel (Frontend 호스팅)
- Supabase (DB·Storage)
- **Vercel** (배포: Git 연동 빌드·배포, 환경 변수 관리)

API:
- REST, `/api/v1` prefix. 인증·엔드포인트 상세는 docs/02_기술정보/API_명세_개요.md 참고.

결제:
