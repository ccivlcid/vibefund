# 1. 제품 개요 (Overview)

## 1.1 제품 정의

VibeFund는 노코드·AI·1인 SaaS 빌더가 자신의 프로젝트를 공개하고,

- 실제 사용자 체험 데이터를 축적하며
- 커뮤니티 기반 피드백을 받고
- 검증 점수를 형성하고
- 상위 프로젝트는 투자 연결까지 이어지는

**한국형 빌더 Pre-Seed 투자 허브 플랫폼**이다.

---

# 2. 문제 정의 (Problem Statement)

## 2.1 한국 시장 문제

- 1인 SaaS 및 AI 기반 창업자 증가
- 초기 단계에서 객관적 검증 데이터 부족
- Pre-Seed 투자 연결 구조 부재
- 기존 VC는 초기 데이터 없는 팀에 소극적
- Product Hunt는 투자 연결 기능이 없음

---

## 2.2 해결 방향

VibeFund는 단순 펀딩 플랫폼이 아닌,

> “시장 반응 데이터를 기반으로 투자 가능성을 가시화하는 플랫폼”

을 목표로 설계된다.

---

# 3. 제품 목표 (Goals)

## 3.1 Phase 1 (핵심 MVP – 검증 + 커뮤니티 중심)

- 회원가입/로그인
- 프로젝트 등록
- 서비스 체험 (iframe 기반)
- 검증 질문 응답 시스템
- 댓글 + 대댓글
- Dev Log (업데이트 기능)
- Vibe Score 1.0 (기본 점수 시스템)
- 랭킹 페이지
- 관리자 검수 기능

---

## 3.2 Phase 2 (투자 연결 구조)

- 프로젝트 등급 시스템 (Bronze / Silver / Gold / Elite)
- 상위 프로젝트 별도 노출 영역
- Investor 전용 접근 영역
- 검증 데이터 대시보드
- 투자 매칭 프로세스

---

# 4. 타겟 유저 (Target Users)

## 4.1 1차 타겟

- 노코드 빌더
- AI 기반 1인 SaaS 창업자
- 자동화 기반 서비스 제작자
- 사이드 프로젝트 운영자

## 4.2 2차 타겟

- 개인 엔젤 투자자
- Pre-Seed 관심 투자자
- 창업 커뮤니티 활동 사용자

---

# 5. 핵심 가치 제안 (Value Proposition)

> 당신의 아이디어를  
> 사용자 반응 데이터로 증명하세요.

VibeFund는  
“아이디어 → 데이터 → 신뢰 → 투자”  
흐름을 만드는 플랫폼이다.

---

# 6. 핵심 기능 요구사항 (Functional Requirements)

---

## 6.1 프로젝트 허브

### F-010 프로젝트 등록

- 프로젝트 제목
- 한 줄 설명 (필수)
- 상세 설명
- 서비스 URL (필수)
- 카테고리 선택
  - AI
  - SaaS
  - Automation
  - No-Code
- 상태 선택
  - Prototype
  - Beta
  - Live
- 썸네일 업로드

---

### F-011 프로젝트 상세 페이지

- iframe 기반 서비스 체험
- 고정 검증 질문 3개:
  1. 이 서비스를 사용할 의향이 있습니까?
  2. 월 얼마까지 지불 가능합니까?
  3. 가장 개선이 필요한 부분은?
- 댓글 기능
- 대댓글 기능 (제작자 답변 강조 UI)
- Dev Log 탭

---

## 6.2 커뮤니티 시스템

### F-030 댓글

- 사용자 댓글 작성
- 대댓글 기능
- 신고 기능
- 관리자 삭제 가능

---

### F-031 Dev Log

- 제작자 업데이트 작성
- 버전 기록
- 개발 과정 공유
- 수정 이력 유지

---

## 6.3 Vibe Score 1.0 (핵심 지표)

Phase 1에서는 단순 가중치 기반 점수 적용.  
**계산 요소(예)**: 서비스 체험 수, 검증 질문 응답(사용 의향·지불 의사), 댓글 수·반응 등. 상세 가중치·공식은 별도 기술 문서에서 정의.

---

## 6.4 랭킹 시스템

- 이번 주 인기 프로젝트
- 상승 중 프로젝트
- 댓글 많은 프로젝트
- 높은 Vibe Score 프로젝트

---

## 6.5 관리자 기능

- 프로젝트 승인 / 반려
- 댓글 신고 처리
- 사용자 차단
- KPI 대시보드

---

# 7. 프로젝트 등급 시스템 (Phase 2)

**Phase 2에서 도입.** 자동 등급 분류:

| 등급 | 기준 |
|------|------|
| Bronze | 기본 등록 |
| Silver | 일정 체험·피드백 달성 |
| Gold | 높은 긍정 반응 |
| Elite | 상위 5~10% |

등급은 자동 계산 기준으로 부여.

---

# 8. 투자 연결 구조 (Phase 2)

## F-070 상위 프로젝트 노출

- Gold 이상 프로젝트 별도 섹션 노출

## F-071 Investor 전용 영역

- 프로젝트 성장 그래프
- 검증 지표 요약
- 빌더 활동 분석
- 댓글 감성 요약 (Phase 3 확장)

---

# 9. 수익 모델 (Revenue Model)

1. 투자 매칭 수수료 (성사 시 2~5%)
2. Investor Pro 구독
3. 프리미엄 노출 상품
4. 데이터 분석 리포트 판매

---

# 10. KPI (Success Metrics)

## 3개월 목표

- 활성 프로젝트 100개
- DAU 300명
- 활성 빌더 30명

## 6개월 목표

- 프로젝트 300개
- 상위 등급 프로젝트 10개
- 첫 투자 매칭 1건

---

# 11. 비기능 요구사항 (Non-Functional Requirements)

- 모바일 반응형 지원
- 페이지 로딩 3초 이내
- 가용성 99% 이상
- CI/CD 자동화
- 관리자 로그 기록
- 에러 모니터링 도입 (Phase 1 완료 후)
