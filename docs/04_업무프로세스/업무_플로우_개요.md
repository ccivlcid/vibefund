# VibeFund 업무 플로우 개요

## 1. 회원 가입·로그인

1. 사용자 → 이메일 가입 또는 OAuth(Google) 선택
2. **백엔드 인증 API** 호출 → 회원 저장(DB)·JWT/세션 발급
3. 로그인 상태 유지 (JWT 또는 세션 쿠키)

## 2. 프로젝트 등록 (제작자)

1. 로그인 → 프로젝트 생성
2. 제목, 한 줄 설명, 상세 설명, 서비스 URL, 카테고리, 썸네일 입력
3. 미리보기 자동 생성 (OpenGraph·iframe)
4. 프로젝트 상태 선택 (Prototype / Beta / Live)

## 3. 펀딩 설정 (제작자)

1. 목표 금액, 마감 기한, 최소 후원 금액 설정
2. 리워드 설정 (베타 접근권, 평생 이용권, 구독 할인 등)
3. 펀딩 오픈

## 4. 후원 (서포터) — 현재

1. 프로젝트 상세 → 서비스 미리보기(iframe) 체험
2. 펀딩 목표·진행률·리워드 안내 확인 (실제 결제 없음)
3. (선택) 댓글 작성, 제작자 답변  
- **추후**: 후원 금액/리워드 선택 → 결제 연동 시 진행률·후원자 수 갱신

## 5. 커뮤니티

- 댓글: 작성 → 제작자 답변
- 프로젝트 업데이트: 제작자 게시 → 서포터 열람

## 6. 유저 페이지 (마이페이지)

1. 로그인 → 유저 페이지(마이페이지) 진입
2. 프로필 보기·수정 (이름, 아바타, 소개 등)
3. 내 프로젝트 목록 확인 → 수정/삭제 진입
4. (선택) 관심·후원 목록, 내 댓글·알림 확인

## 7. 관리자 페이지

1. 관리자 계정 로그인 → 관리자 전용 URL 진입
2. 대시보드: KPI(회원 수, 프로젝트 수, 검수 대기 등) 확인
3. 회원 관리: 목록·검색·상세, 정지 등
4. 프로젝트 검수: 목록·승인/반려/비공개, 반려 사유 입력
5. 댓글·신고: 신고 접수 건 조회·삭제/유지 처리

---

## 보완: 예외·부가 플로우

### 7. 결제 실패·환불 (추후 결제 도입 시)

- 결제 실패 시: 에러 메시지 표시, 재시도 안내
- 환불: 운영 정책에 따라 환불 처리 → Pledge 상태 업데이트

### 8. 프로젝트 검수(운영)

- **관리자 페이지**에서 수행. 등록 후 검수(필요 시): 부적절 콘텐츠·URL 검증 → 승인/반려
- 반려 시: 제작자에게 사유 안내, 수정 후 재제출

### 9. 권한

- 프로젝트 수정/삭제: 제작자 본인만
- 댓글 삭제: 작성자 또는 제작자(본 프로젝트 댓글)
- 펀딩 마감 후: 목표 금액·리워드 변경 제한 (정책으로 정의)

---

## 10. 배포·릴리스 (Vercel)

1. 개발자가 코드 푸시 → Vercel이 Git 변경 감지
2. Vercel 자동 빌드·배포 (Preview 브랜치별, Production은 main 등 설정에 따름)
3. 배포 실패 시: Vercel 빌드 로그 확인, 수정 후 재푸시 또는 이전 배포로 롤백
