# VibeFund 기술 스택 (PRD 기준)

**적용 결정**: 결제 기능 현재 미적용, DB는 Supabase PostgreSQL 단일 사용, Redis 미사용, **인증은 Supabase Auth 미사용·백엔드 인증 API 사용**. **CI/CD는 Vercel Git 연동**, **리포지터리는 모노레포** 구조(apps/web + packages/shared 등).

## 1. 리포지터리·모노레포

| 항목 | 기술 | 비고 |
|------|------|------|
| 구조 | **모노레포** | 단일 리포에서 앱·공용 패키지 관리. [01_아키텍처/모노레포_구조.md](../01_아키텍처/모노레포_구조.md) 참고 |
| 워크스페이스 | pnpm workspace | apps/*, packages/* |
| 빌드·태스크 | Turborepo (권장) | 캐시·의존성 순서 관리. 루트에서 `turbo run build` 등 |

## 2. Frontend

| 항목 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | Next.js | SSR/SSG, API Routes 활용 |
| 스타일 | Tailwind CSS | 반응형, 유지보수 용이 |

## 3. Backend · DB

| 항목 | 기술 | 비고 |
|------|------|------|
| API | Next.js API Routes + Supabase Client | REST API, **인증 API(회원가입/로그인/로그아웃·JWT/세션)** 포함 |
| DB | **Supabase (PostgreSQL)** | 단일 DB. 인증 시 users 등 테이블 사용, Supabase Auth 미사용 |
| 캐시 | 없음 (Redis 미사용) | 필요 시 메모리 또는 DB 쿼리 최소화로 대체 |

## 4. 인프라

| 항목 | 기술 | 비고 |
|------|------|------|
| **CI/CD** | **Vercel Git 연동** | Git 푸시 기반 빌드·배포. Vercel 대시보드에서 환경 변수·빌드 설정 관리 |
| Frontend 호스팅 | Vercel | Next.js 배포 최적화 |
| DB | **Supabase** | PostgreSQL·Storage 등 (인증은 백엔드 API에서 처리) |

## 5. 결제

- **현재**: 결제 기능 미구현. 펀딩 목표·진행률·리워드 설정 및 표시만 제공.
- **추후**: Stripe 등 도입 시 별도 문서 반영.

## 6. 비기능 요구사항 반영

- **모바일 반응형**: Tailwind 반응형 유틸
- **로딩 3초 이내**: CDN(Vercel), Supabase 연결 최적화
- **가용성 99%+**: Vercel·Supabase 관리형 서비스 활용

## 7. 보완 사항

### 7.1 버전·호환성 (권장)

- Node.js: LTS 18+ (Next.js 호환)
- Supabase: 제공 PostgreSQL 15 호환

### 7.2 환경·보안

- **환경 변수**: DB 연결은 `DATABASE_URL`(PostgreSQL 연결 문자열) 사용. Supabase 호스트: `db.oxyjutbyfejuhewugyly.supabase.co:5432/postgres`. 상세 형식·설정 방법은 [환경변수_설정.md](./환경변수_설정.md) 참고. 비밀번호·키는 `.env`(로컬) 또는 Vercel 환경 변수(배포)에만 두고 저장소에 커밋 금지.
- **인증**: 백엔드 인증 API에서 이메일 가입/로그인·OAuth(Google) 처리 후 JWT 또는 세션 쿠키 발급

### 7.3 배포

- Frontend: Vercel Git 연동 자동 배포
- DB: Supabase 대시보드에서 마이그레이션 또는 SQL 에디터 실행
