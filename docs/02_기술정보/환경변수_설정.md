# VibeFund 환경 변수 설정

애플리케이션·CI/CD에서 사용하는 환경 변수 정리입니다.  
**비밀번호·API 키 등은 Git에 커밋하지 않고**, `.env`(로컬) 또는 Vercel 환경 변수(배포)에서만 관리하세요.

---

## 1. DATABASE_URL (필수)

Supabase PostgreSQL 연결용 연결 문자열입니다.

### 1.1 형식

```
postgresql://<USER>:<PASSWORD>@<HOST>:<PORT>/<DATABASE>
```

### 1.2 현재 프로젝트 기준 값 (구성만 참고)

| 항목 | 값 |
|------|-----|
| **HOST** | `db.oxyjutbyfejuhewugyly.supabase.co` |
| **PORT** | `5432` |
| **DATABASE** | `postgres` |
| **USER** | `postgres` |
| **PASSWORD** | (Supabase 대시보드 또는 팀에서 발급한 비밀번호. 이 문서에 기입하지 말 것) |

### 1.3 설정 방법

- **로컬 개발**: 프로젝트 루트 또는 `apps/web`에 `.env` 파일을 만들고 아래 한 줄 추가.  
  **`.env`는 반드시 `.gitignore`에 포함하여 저장소에 올리지 마세요.**

  ```
  DATABASE_URL=postgresql://postgres:<비밀번호>@db.oxyjutbyfejuhewugyly.supabase.co:5432/postgres
  ```

- **배포 (Vercel)**: Vercel 프로젝트 설정 → Environment Variables에 `DATABASE_URL` 이름으로 동일 형식의 값을 등록.

- **참고**: 실제 비밀번호가 포함된 전체 연결 문자열은 팀 내부 비밀 채널로만 공유하고, 문서·코드 저장소에는 넣지 않습니다.

### 1.4 .env.example (커밋용 예시)

저장소에는 비밀 없이 예시만 둡니다. 복사 후 `.env`로 저장하고 실제 값으로 채우세요.

```
# .env.example — 복사 후 .env 로 저장하고 실제 값 입력
DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@db.oxyjutbyfejuhewugyly.supabase.co:5432/postgres
```

---

## 2. 기타 환경 변수 (추가 예정)

- **JWT 시크릿**: 인증 API에서 사용. 예: `JWT_SECRET`
- **OAuth (Google)**: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`
- **Supabase**: `NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` (선택, Supabase Client 사용 시)
- **Vercel**: 배포 시 Vercel 대시보드에서 `DATABASE_URL` 등 주입
