# VibeFund 모노레포 구조

단일 리포지터리에서 프론트엔드·공용 패키지를 함께 관리하는 **모노레포(Monorepo)** 구조를 사용합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 목적 | 앱(웹)·공용 코드·타입·유틸을 한 리포에서 버전 관리, 빌드·테스트 일원화 |
| 도구 | **pnpm workspace** 또는 **Turborepo** (캐시·태스크 오케스트레이션). 팀 선택에 따라 Nx 대안 가능 |
| 배포 단위 | `apps/web`(Next.js)를 Vercel에 배포. API는 동일 앱 내 API Routes 또는 추후 `apps/api` 분리 가능 |

---

## 2. 디렉터리 구조 (권장)

```
vibeFund/
├── apps/
│   └── web/                 # Next.js 앱 (프론트 + API Routes)
│       ├── app/
│       ├── pages/
│       ├── src/
│       ├── package.json
│       └── next.config.js
├── packages/
│   ├── shared/              # 공용 타입·상수·유틸 (apps/web에서 import)
│   │   ├── src/
│   │   └── package.json
│   ├── db/                  # (선택) DB 클라이언트·스키마 타입
│   │   ├── src/
│   │   └── package.json
│   └── eslint-config/       # (선택) 공용 ESLint 설정
├── package.json             # 루트 workspace 정의
├── pnpm-workspace.yaml      # pnpm workspace 루트
├── turbo.json               # (Turborepo 사용 시) 파이프라인 정의
└── docs/                    # 프로젝트 문서 (현재 docs/)
```

- **apps/web**: 사용자·관리자 UI + Next.js API Routes(인증·프로젝트·펀딩·댓글 등). Vercel 배포 대상.
- **packages/shared**: API 요청/응답 타입, 상수, 공용 유틸. `apps/web`에서 `@vibefund/shared` 등으로 참조.
- **packages/db**: (선택) Supabase 클라이언트 래퍼, 생성된 타입. 여러 앱에서 사용할 때 유리.

## 3. 워크스페이스 설정 예시

### 3.1 pnpm workspace

**pnpm-workspace.yaml** (루트):

```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

루트 **package.json**:

```json
{
  "name": "vibefund",
  "private": true,
  "scripts": {
    "build": "turbo run build",
    "dev": "turbo run dev",
    "lint": "turbo run lint"
  },
  "devDependencies": {
    "turbo": "^2.x"
  },
  "packageManager": "pnpm@9.x"
}
```

### 3.2 Turborepo (선택)

**turbo.json** (루트):

```json
{
  "$schema": "https://turbo.build/schema.json",
  "pipeline": {
    "build": { "dependsOn": ["^build"], "outputs": [".next/**", "dist/**"] },
    "dev": { "cache": false, "persistent": true },
    "lint": {}
  }
}
```

- `apps/web` 빌드 시 `packages/shared`(및 기타 의존 패키지)가 먼저 빌드되도록 `dependsOn: ["^build"]` 사용.

---

## 4. Vercel 배포와의 연동

- **빌드 경로**: 모노레포 루트에서 실행. Turborepo 사용 시 `turbo run build` 또는 `pnpm run build`로 전체(또는 변경된 패키지만) 빌드.
- **배포 대상**: `apps/web` 빌드 산출물(.next)을 Vercel에 배포. Vercel 프로젝트에서 루트 디렉터리를 리포지터리 루트 또는 `apps/web`으로 설정.
- **의존성 설치**: Vercel 빌드 시 루트에서 `pnpm install` 후 빌드. `pnpm install --frozen-lockfile` 권장.

---

## 5. 네이밍·패키지 참조

- **패키지 이름**: `@vibefund/web`, `@vibefund/shared`, `@vibefund/db` 등 스코프 사용 권장.
- **내부 참조**: `apps/web`의 package.json에 `"@vibefund/shared": "workspace:*"` 형태로 의존성 추가.

---

## 6. 확장 시

- **apps/api**: FastAPI/Node 등 별도 API 서버가 필요하면 `apps/api` 추가. 모노레포 내에서 `packages/shared`, `packages/db` 공유.
- **packages/ui**: 공용 React 컴포넌트 라이브러리 분리 시 추가.
