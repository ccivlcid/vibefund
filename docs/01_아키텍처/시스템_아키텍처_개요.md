# VibeFund 시스템 아키텍처 개요

## 1. 개요

VibeFund는 **노코드 창업자용 웹서비스 전용 검증형 펀딩 플랫폼**으로,  
**모노레포** 단일 리포지터리에서 프론트엔드·API·공용 패키지를 관리하고, 프론트엔드·백엔드·DB를 분리한 웹 애플리케이션 구조를 가집니다.  
**현재 MVP에서는 결제 기능을 제외**하며, DB는 Supabase PostgreSQL 단일 사용, Redis 미사용입니다.  
**인증은 Supabase Auth를 쓰지 않고, 백엔드에서 구현한 인증 API**를 사용합니다.  
리포지터리 구조·워크스페이스는 [모노레포_구조.md](./모노레포_구조.md) 참고.

## 2. 구성 요소

```
              [Vercel Git 연동]
                          │
    Git 푸시 → 빌드·배포 (자동)
                          │
                          ▼
[사용자] → [Frontend (Next.js)] → [Backend API] → [Supabase PostgreSQL]
                │                        │
                │   (모노레포: apps/web + packages/shared 등)
                ▼                        ▼
           [Vercel]              인증 API · 비즈니스 로직
```

| 레이어 | 기술 | 역할 |
|--------|------|------|
| **CI/CD** | **Vercel Git 연동** | Git 푸시 기반 빌드·배포, Vercel 환경 변수로 시크릿 관리 |
| Frontend | Next.js, Tailwind CSS | SPA/SSR, UI 렌더링 |
| Backend | Next.js API Routes 등 | **인증 API(회원가입/로그인/로그아웃·JWT/세션)** + REST API, 비즈니스 로직 |
| DB | **Supabase (PostgreSQL)** | 영구 데이터 저장(users 등). 인증은 백엔드 API가 DB 사용 |
| Infra | Vercel (FE), Supabase (DB) | 호스팅 |

- **인증**: Supabase Auth 미사용. 백엔드 인증 API에서 이메일·OAuth(Google) 처리 후 JWT 또는 세션 쿠키 발급.
- **Redis 미사용**: 세션/토큰은 DB 또는 쿠키(JWT)로 처리. 캐시가 필요하면 메모리/DB로 한정.
- **결제**: 현재 범위 제외. 추후 Stripe 등 도입 시 반영 예정.

## 3. 주요 흐름

- **회원/인증**: 이메일·OAuth(Google) → **백엔드 인증 API** → 회원 저장(DB)·JWT/세션 발급
- **프로젝트 등록**: URL/메타정보 수집 → 미리보기(iframe)·썸네일 생성
- **펀딩(현재)**: 목표·마감·리워드 설정 및 진행률 표시만. 실제 결제 없음.

## 4. 보안·성능 요구사항 (PRD 기준)

- 페이지 로딩: 3초 이내
- 가용성: 99% 이상 목표
- 결제 도입 시: PCI 준수(Stripe 위임) 등 별도 정리

## 5. API·통신 원칙 (보완)

- **인증**: 백엔드 인증 API에서 발급한 JWT 또는 세션 쿠키, HTTPS 필수
- **API**: RESTful 규칙, 버전 prefix `/api/v1/`. 엔드포인트·요청/응답 상세는 [02_기술정보/API_명세_개요.md](../02_기술정보/API_명세_개요.md) 참고
- **에러**: 일관된 HTTP 상태 코드·에러 메시지 포맷 (동일 문서 참고)

## 6. CI/CD (Vercel)

- **배포**: Git 푸시 시 Vercel이 자동 빌드·배포. Vercel 대시보드에서 환경(Production/Preview)·환경 변수 관리.
- **롤백**: Vercel 대시보드에서 이전 배포로 롤백 가능.

## 7. 장애·예외 대응

- iframe 미리보기 실패: URL 유효성 검사, 샌드박스 속성으로 XSS 방지
- Supabase 장애: 대시보드·상태 페이지 확인, 로그·알림 연동
- 배포 실패: Vercel 빌드 로그 확인, 수정 후 재배포 또는 이전 배포로 롤백
