## 3. ERD (Entity Relationship Diagram)

```
┌──────────────────┐          ┌──────────────────────────────────┐
│      users       │          │            projects               │
├──────────────────┤          ├──────────────────────────────────┤
│ PK  id (UUID)    │──────┐   │ PK  id (UUID)                    │
│     email        │      │   │ FK  user_id ──────────────────── │──┐
│     name         │      └──▶│     title                        │  │
│     avatar_url   │    1:N   │     short_description            │  │
│     bio          │          │     description                  │  │
│     provider     │          │     service_url                  │  │
│     role         │          │     category                     │  │
│  (user/admin)    │          │     thumbnail_url                │  │
│     created_at   │          │     status (Prototype/Beta/Live) │  │
│     updated_at   │          │     approval_status              │  │
└──────────────────┘          │     deleted_at (소프트 삭제)      │  │
        │                     │     created_at                   │  │
        │ 1:N (후원자)         │     updated_at                   │  │
        │                     └──────────────┬───────────────────┘  │
        │                                    │                       │
        │         ┌──────────────────────────┼───────────────────────┤
        │         │                          │                       │
        │         ▼ 1:1                      ▼ 1:N                   │
        │  ┌─────────────────┐      ┌────────────────────┐          │
        │  │    fundings      │      │      rewards        │          │
        │  ├─────────────────┤      ├────────────────────┤          │
        │  │ PK id (UUID)    │      │ PK id (UUID)        │          │
        │  │ FK project_id ──│      │ FK project_id       │          │
        │  │    goal_amount  │      │    name             │          │
        │  │    deadline     │      │    description      │          │
        │  │ min_pledge_amt  │      │    amount           │          │
        │  │    created_at   │      │    type             │          │
        │  │    updated_at   │      │  (beta/lifetime/    │          │
        │  └─────────────────┘      │   subscription)    │          │
        │                           │    created_at       │          │
        │                           │    updated_at       │          │
        │                           └────────────────────┘          │
        │                                                            │
        │                           ┌────────────────────────┐      │
        │                           │       pledges           │      │
        │                           ├────────────────────────┤      │
        └───────────────────────┐   │ PK id (UUID)            │      │
             1:N (후원)          └──▶│ FK user_id              │      │
                                    │ FK project_id ──────────│──────┘
                                    │ FK reward_id (nullable) │
                                    │    amount               │
                                    │    stripe_payment_id    │
                                    │    status               │
                                    │    created_at           │
                                    └────────────────────────┘

┌──────────────────────────────────────────────────┐
│                   comments                        │
├──────────────────────────────────────────────────┤
│ PK id (UUID)                                     │
│ FK project_id → projects.id                      │
│ FK user_id    → users.id                         │
│ FK parent_id  → comments.id (nullable, 대댓글)   │◀──┐
│    body                                          │   │ self-reference
│    created_at                                    │   │ (대댓글)
│    updated_at                                    │───┘
└──────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────┐
│                   updates                         │
├──────────────────────────────────────────────────┤
│ PK id (UUID)                                     │
│ FK project_id → projects.id                      │
│ FK user_id    → users.id                         │
│    title                                         │
│    body                                          │
│    created_at                                    │
│    updated_at                                    │
└──────────────────────────────────────────────────┘

관계 요약:
  users     1 ──── N  projects   (제작자)
  users     1 ──── N  pledges    (후원자)
  projects  1 ──── 1  fundings   (펀딩 설정)
  projects  1 ──── N  rewards    (리워드)
  projects  1 ──── N  pledges    (후원 내역)
  projects  1 ──── N  comments   (댓글)
  projects  1 ──── N  updates    (업데이트)
  comments  1 ──── N  comments   (대댓글, self-reference)
```

## 4. 사용자 플로우 다이어그램

### 4.1 제작자 플로우

```
시작
  │
  ▼
[회원가입 / 로그인]
  │ 이메일 또는 Google OAuth
  ▼
[대시보드 / 홈]
  │
  ├─────────────────────────────────────┐
  │                                     │
  ▼                                     ▼
[프로젝트 생성]                    [유저 페이지(마이페이지)]
  │                                     │
  │ 제목, 설명, 서비스 URL,              ├── 프로필 수정
  │ 카테고리, 썸네일 입력               ├── 내 프로젝트 목록
  │                                     ├── 관심/후원 목록
  ▼                                     └── 내 댓글 활동
[미리보기 자동 생성]
  │ OpenGraph 썸네일 수집
  │ iframe 미리보기 렌더
  ▼
[프로젝트 상태 선택]
  │ Prototype / Beta / Live
  ▼
[펀딩 설정]
  │ 목표 금액, 마감일, 최소 후원액
  ▼
[리워드 설정]
  │ 베타 접근권, 평생 이용권, 구독 할인
  ▼
[프로젝트 공개]
  │
  ▼
[검수 대기] ──── 관리자 검수 ────▶ [승인] ──▶ [공개 노출]
                                       │
                                       └──▶ [반려] ──▶ [제작자에 사유 안내]
                                                           │
                                                           └──▶ [수정 후 재제출]
  │
  ├── 댓글 답변 (서포터 댓글에 답변)
  └── 업데이트 게시 (프로젝트 소식 공유)
```

### 4.2 서포터 플로우

```
시작
  │
  ▼
[홈 / 프로젝트 탐색]
  │ 카테고리·상태 필터, 검색
  ▼
[프로젝트 상세 페이지]
  │
  ├── [서비스 미리보기 (iframe)]
  │     실제 서비스 체험
  │
  ├── [펀딩 정보 확인]
  │     목표 금액, 진행률, 마감일
  │
  ├── [리워드 목록 확인]
  │     베타 접근권, 평생 이용권 등
  │
  ├── [댓글 작성]     ← (현재: 관심 표현)
  │
  └── [후원하기]      ← (추후: 결제 연동 후 활성화)
        │
        ▼
      [리워드 선택 → 결제 처리 → Pledge 생성]  (Phase 2)
```

### 4.3 관리자 플로우

```
[관리자 로그인]
  │ role=admin 검증
  ▼
[관리자 대시보드]
  │ KPI 요약: 회원 수, 프로젝트 수, 검수 대기 수
  │
  ├──────────────────────────────────────────┐
  │                                           │
  ▼                                           ▼
[회원 관리]                              [프로젝트 검수]
  │                                           │
  ├── 목록·검색                              ├── 검수 대기 목록
  ├── 계정 정지 처리                         ├── 승인 처리
  └── 권한 변경 (admin 부여)                 ├── 반려 (사유 입력)
                                             └── 비공개 처리
  ▼
[댓글·신고 관리]
  │
  ├── 신고 접수 목록 조회
  └── 댓글 삭제 / 유지 처리
```

## 5. 인증 시퀀스 다이어그램

### 5.1 이메일 로그인

```
클라이언트       Next.js API Route      Supabase DB
    │                   │                    │
    │─── POST /api/v1/auth/login ──────────▶ │
    │    { email, password }                  │
    │                   │                    │
    │                   │── SELECT users ───▶│
    │                   │   WHERE email=?    │
    │                   │◀── user row ───────│
    │                   │                    │
    │                   │ bcrypt.compare()   │
    │                   │ (password vs hash) │
    │                   │                    │
    │                   │ jwt.sign(payload)  │
    │                   │                    │
    │◀── 200 { user, token } ────────────────│
    │    Set-Cookie: token=JWT;              │
    │    HttpOnly; Secure; SameSite=Strict   │
```

### 5.2 인증 미들웨어 (보호된 API)

```
클라이언트       미들웨어 (withAuth)    API Handler
    │                   │                    │
    │─── GET /api/v1/users/me ─────────────▶│
    │    Cookie: token=JWT                   │
    │                   │                    │
    │                   │ jwt.verify(token)  │
    │                   │                    │
    │         ┌─────────┴─────────┐          │
    │         │ 성공              │ 실패      │
    │         ▼                  ▼           │
    │    req.user = payload   401 Unauthorized
    │         │                              │
    │         └─── 핸들러 호출 ─────────────▶│
    │                                        │ DB 조회
    │◀── 200 { data: user } ─────────────────│
```

### 5.3 Google OAuth

```
클라이언트       API Route          Google OAuth       Supabase DB
    │                │                   │                  │
    │── GET /auth/ ─▶│                   │                  │
    │   oauth/google │                   │                  │
    │◀── 302 ────────│                   │                  │
    │    accounts.google.com             │                  │
    │                                    │                  │
    │─── [Google 동의 화면] ─────────────│                  │
    │                                    │                  │
    │◀── redirect: /oauth/callback ──────│                  │
    │    ?code=AUTH_CODE                 │                  │
    │                │                   │                  │
    │── GET /callback?code=... ─────────▶│                  │
    │                │                   │                  │
    │                │── POST /token ────▶│                 │
    │                │◀── access_token ──│                  │
    │                │                   │                  │
    │                │── GET /userinfo ──▶│                 │
    │                │◀── { sub, email } │                  │
    │                │                   │                  │
    │                │── UPSERT users ────────────────────▶ │
    │                │◀── user ──────────────────────────── │
    │                │                                      │
    │                │ jwt.sign(payload)                    │
    │◀── 302 → /dashboard ──────────────────────────────── │
    │    Set-Cookie: token=JWT                              │
```
