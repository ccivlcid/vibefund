# VibeFund UI/UX 가이드 개요

## 1. 포지셔닝 (PRD §13)

> 노코드 창업자를 위한  
> 웹서비스 전용 검증형 펀딩 플랫폼

## 2. 타겟에 따른 UX 원칙

- **제작자(비개발자)**: 단계가 명확한 프로젝트 등록·펀딩 설정, URL만 넣으면 미리보기 자동 생성
- **서포터**: 실제 서비스 체험(iframe) → 후원 결정 구조로 신뢰감 제공
- **공통**: 모바일 반응형, 3초 이내 로딩 (PRD 비기능 요구사항)

## 3. 핵심 화면 (MVP)

| 화면 | 목적 |
|------|------|
| 메인/탐색 | 프로젝트 목록, 카테고리, 진행 중인 펀딩 노출 |
| 프로젝트 상세 | 썸네일·설명·미리보기(iframe)·펀딩 진행률·후원 CTA·댓글 |
| 프로젝트 등록 | 단계별 입력(제목, 설명, URL, 카테고리, 썸네일) |
| 펀딩 설정 | 목표 금액, 마감일, 최소 후원액, 리워드 |
| 결제 | 현재 미제공. 추후 결제 도입 시 전용 화면·완료/실패 피드백 |
| **유저 페이지(마이페이지)** | 로그인 사용자 전용. 프로필 보기·수정, 내 프로젝트 목록, 관심/후원 목록, 내 댓글·알림 |
| 프로필(공개) | 제작자 정보, 내 프로젝트 (후원 내역은 결제 도입 후) |
| **관리자 페이지** | 관리자 전용. 대시보드(KPI), 회원 관리, 프로젝트 검수(승인/반려), 댓글·신고 관리 |

## 4. 추후 정의 예정

- 컬러·폰트·spacing (Tailwind 기반 디자인 토큰)
- 컴포넌트 라이브러리 규칙
- 접근성(키보드·스크린리더) 체크리스트

## 5. 보완: 상태별 UX

| 상태 | 권장 처리 |
|------|------------|
| 로딩 | 스켈레톤 또는 스피너, 3초 초과 시 메시지 안내 |
| 빈 상태 | "아직 프로젝트가 없어요", CTA로 등록 유도 |
| 에러 | 네트워크·API 에러 시 명확한 메시지·재시도 버튼 (결제 실패는 추후) |
| 성공 | 등록 완료·업데이트 완료 시 축하 메시지·다음 단계 안내 |
