# 1. UI/UX 핵심 원칙

## 1.1 신뢰 우선 설계

- 감성보다 **데이터 우선 노출**
- 프로젝트 카드에서 5초 안에 가치 판단 가능해야 함
- 제작자 활동성(업데이트/답변) 강조

---

## 1.2 5초 판단 카드 구조

프로젝트 리스트에서 즉시 보이는 요소:

- 썸네일
- 제목
- 카테고리 뱃지
- 상태 (Prototype / Beta / Live)
- Vibe Score
- 체험 수
- 댓글 수
- 최근 업데이트 날짜
- CTA: [체험하기] [피드백]

---

## 1.3 커뮤니티는 “딜 소싱 도구”

- 자유 토론이 아닌 **검증 중심 구조**
- 댓글 전에 정형 질문 응답 유도
- 제작자 답변은 강조 표시

# 3. 주요 화면 UX 설계

---

# 3.1 Discover (홈)

## 목적
- 검증 가치 있는 프로젝트 발견
- 투자 관점에서 신뢰 지표 노출

## 구성

### 상단
- 검색창
- 필터
  - 카테고리
  - 상태
  - 정렬 (Trending / Vibe Score / 댓글순 / 최신순)

### 중단
- Today’s Rising
- This Week Top

### 하단
- 프로젝트 카드 그리드

---

# 3.2 프로젝트 카드 구성

- 썸네일
- 제목
- 카테고리 뱃지
- 상태 뱃지
- Vibe Score
- 체험 수
- 피드백 수
- 최근 업데이트
- CTA 버튼 2개

---

# 3.3 프로젝트 상세 페이지

## 탭 구조

- Overview
- Demo
- Validation
- Updates
- Comments

---

## Overview

- 프로젝트 헤더
- 제작자 정보
- 신뢰 위젯:
  - Vibe Score
  - 체험 수
  - 긍정 응답률
  - 최근 업데이트

---

## Demo

- iframe 기반 체험
- 새 창 열기 버튼
- 보안 안내 문구

---

## Validation (핵심)

### 고정 질문 3개

1. 이 서비스를 사용할 의향이 있습니까?
2. 월 얼마까지 지불 가능합니까?
3. 가장 개선이 필요한 부분은?

### 응답 방식

- 선택형 + 자유 입력
- 응답 후 집계 결과 즉시 표시

---

## Updates (Dev Log)

- 타임라인 형태
- 업데이트 날짜
- 변경 내용
- 사용자 반응(간단 좋아요)

---

## Comments

- 댓글 작성
- 대댓글 기능
- 제작자 답변 강조 UI
- 신고 버튼
- 관리자 삭제 가능

---

# 3.4 Launch (프로젝트 등록)

## Step 1: 기본 정보

- 제목
- 한 줄 설명
- URL
- 카테고리 (필수)
- 상태 (필수)

## Step 2: 검증 설정

- 원하는 피드백 유형 선택
- 초기 질문 설정

---
