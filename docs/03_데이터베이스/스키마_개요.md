# VibeFund 스키마 개요

PRD v2.0 기능 요구사항을 반영한 테이블·컬럼 요약입니다.  
**운영**: Supabase PostgreSQL. 실제 DDL은 Supabase SQL 에디터 또는 마이그레이션으로 관리합니다.  
**결제 미적용**: pledges 테이블은 Phase 2 또는 추후 결제 도입 시 사용.

## 1. users

- id, email, name, avatar_url, provider (oauth), **role** (user / admin — 관리자 페이지 접근용), created_at, updated_at

## 2. projects

- id, user_id, title, short_description, description, service_url, **category** (AI/SaaS/Automation/No-Code), thumbnail_url, **status** (Prototype/Beta/Live), **approval_status** (pending/approved/rejected — 관리자 검수용), created_at, updated_at, deleted_at (선택)

## 3. fundings

- id, project_id, goal_amount, deadline, min_pledge_amount, created_at, updated_at

## 4. rewards

- id, project_id, name, description, amount, type (beta/lifetime/subscription_discount), created_at, updated_at

## 5. pledges

- id, project_id, user_id, amount, reward_id (nullable), stripe_payment_id, status, created_at, updated_at

## 6. comments

- id, project_id, user_id, parent_id (nullable, 대댓글), body, created_at, updated_at

## 7. updates

- id, project_id, user_id, title, body, created_at, updated_at (Dev Log)

## 8. verification_responses (PRD v2.0 — 검증 질문 응답)

- id, project_id, user_id, **question_key** (q1_use_intent / q2_monthly_pay / q3_improvement 등), **answer** (텍스트 또는 코드화된 값), created_at  
- 고정 3문항: (1) 사용 의향 (2) 월 지불 가능 금액 (3) 가장 개선이 필요한 부분. Vibe Score 계산·집계에 사용.

## 9. comment_reports (댓글 신고)

- id, comment_id, reporter_user_id, reason (선택), status (pending/resolved_dismissed/resolved_deleted), created_at, updated_at  
- F-054 관리자 신고 처리용.

## 10. vibe_scores (PRD v2.0 — Vibe Score 1.0)

- id, project_id, **score** (숫자), **calculated_at**, 메타(선택)  
- 또는 매 요청 시 뷰/집계 쿼리로 계산. Phase 1에서는 단순 가중치(체험 수·검증 응답·댓글 등) 반영.

**랭킹**: 별도 테이블 없이 projects + vibe_scores + comments/updates 집계로 랭킹 쿼리 또는 materialized view 구성 가능.

## 11. AI 이사회 (AI Board — 사전심사 시스템)

### 11.1 ai_board_members (멤버 풀)

- id (UUID), **key** (text, unique. 예: buffett, jobs, karpathy, gates, musk, lynch, thiel), **display_name** (표시명), **perspective** (관점 한 줄), default_weight_percent (0~100, 기본 가중치), is_active (boolean), sort_order (int), created_at, updated_at  
- 운영자가 멤버 풀에 유명인 추가·비활성화 가능.

### 11.2 project_ai_board_selections (프로젝트별 유저 선택)

- id (UUID), **project_id** (FK → projects), **ai_board_member_id** (FK → ai_board_members), weight_percent (nullable, 해당 프로젝트에서의 가중치; null이면 멤버 기본값 사용), created_at  
- **유저가 “포함”한 멤버만** row 존재. 제외한 멤버는 row 없음. unique(project_id, ai_board_member_id).

### 11.3 ai_board_reports (심사 결과)

- id (UUID), **project_id** (FK → projects), **overall_score** (numeric 0~10), **grade** (enum: A/B/C/D), strengths (jsonb 배열), risks (jsonb 배열), improvements (jsonb 배열), **calculated_at**, created_at  
- 프로젝트당 여러 회차 가능(이력). 최신 1건을 “현재 심사 결과”로 사용 권장.

### 11.4 ai_board_report_scores (멤버별 점수·피드백)

- id (UUID), **ai_board_report_id** (FK → ai_board_reports), **ai_board_member_id** (FK → ai_board_members), **score** (numeric 0~10), feedback (text), details (jsonb, 선택), created_at  
- 멤버 추가 시에도 컬럼 변경 없이 확장 가능. unique(ai_board_report_id, ai_board_member_id).

---

## 12. project_votes (따봉/비추 — Vibe 관련 투표)

- id (UUID), **project_id** (FK → projects), **user_id** (FK → users), **vote** (enum: up / down), created_at, updated_at  
- **unique(project_id, user_id)**: 한 유저당 프로젝트당 1표. 추천(따봉) up / 비추천 down.  
- Vibe Score 산정 시 (따봉 수 − 비추 수) 반영 가능.

---

## 13. community_posts (Community 게시판)

- id (UUID), **board** (text: 'discussion' | 'learning'), **user_id** (FK → users), **title**, **body**, created_at, updated_at  
- **discussion**: 자유 토론, **learning**: 창업 학습. 목록/상세 API 및 Community 페이지에서 사용.

---

## 13-1. community_post_comments (Community 게시글 댓글)

- id (UUID), **post_id** (FK → community_posts), **user_id** (FK → users), **parent_id** (FK → self, 대댓글), **body**, created_at, updated_at

---

## 14. chat_rooms / chat_messages (Community 채팅)

- **chat_rooms**: id (UUID), **key** (text UNIQUE, 예: discussion, learning), **name**, created_at  
- **chat_messages**: id (UUID), **room_id** (FK → chat_rooms), **user_id** (FK → users), **body**, created_at  
- 채팅방 목록·방별 메시지 조회/전송 API 및 Community 채팅 UI에서 사용.

## 보완: 제약·인덱스 권장

| 테이블 | PK | FK | 인덱스 권장 |
|--------|----|----|-------------|
| users | id (UUID) | - | email UNIQUE |
| projects | id (UUID) | user_id → users | user_id, status, category, created_at |
| fundings | id (UUID) | project_id → projects | project_id UNIQUE |
| rewards | id (UUID) | project_id → projects | project_id |
| pledges | id (UUID) | project_id, user_id | project_id, user_id, created_at |
| comments | id (UUID) | project_id, user_id, parent_id | project_id, created_at |
| updates | id (UUID) | project_id, user_id | project_id, created_at |
| verification_responses | id (UUID) | project_id, user_id | project_id, question_key, created_at |
| comment_reports | id (UUID) | comment_id, reporter_user_id | comment_id, status |
| vibe_scores | id (UUID) | project_id | project_id, calculated_at |
| ai_board_members | id (UUID) | - | key UNIQUE, is_active, sort_order |
| project_ai_board_selections | id (UUID) | project_id, ai_board_member_id | project_id, (project_id, ai_board_member_id) UNIQUE |
| ai_board_reports | id (UUID) | project_id | project_id, calculated_at desc |
| ai_board_report_scores | id (UUID) | ai_board_report_id, ai_board_member_id | ai_board_report_id, (ai_board_report_id, ai_board_member_id) UNIQUE |
| project_votes | id (UUID) | project_id, user_id | project_id, (project_id, user_id) UNIQUE, vote |
| community_posts | id (UUID) | user_id | board, created_at desc, user_id |
| community_post_comments | id (UUID) | post_id, user_id, parent_id | post_id, (post_id, created_at) |
| chat_rooms | id (UUID) | - | key UNIQUE |
| chat_messages | id (UUID) | room_id, user_id | room_id, (room_id, created_at desc) |

- **민감 정보**: 카드 번호·CVC 등 저장 금지. 추후 결제 도입 시 결제 대행사 payment_id만 저장.
- **소프트 삭제**: projects 등 필요 시 `deleted_at` 컬럼 추가 검토.
