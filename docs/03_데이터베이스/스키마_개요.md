# VibeFund 스키마 개요

PRD 기능 요구사항(F-001 ~ F-031)을 반영한 테이블·컬럼 요약입니다.  
**운영**: Supabase PostgreSQL. 실제 DDL은 Supabase SQL 에디터 또는 마이그레이션으로 관리합니다.  
**결제 미적용**: pledges 테이블은 추후 결제 도입 시 사용. 현재는 스키마만 정의하거나 생략 가능.

## 1. users

- id, email, name, avatar_url, provider (oauth), **role** (user / admin — 관리자 페이지 접근용), created_at, updated_at

## 2. projects

- id, user_id, title, short_description, description, service_url, category, thumbnail_url, status (Prototype/Beta/Live), **approval_status** (pending/approved/rejected — 관리자 검수용, 선택), created_at, updated_at

## 3. fundings

- id, project_id, goal_amount, deadline, min_pledge_amount, created_at, updated_at

## 4. rewards

- id, project_id, name, description, amount, type (beta/lifetime/subscription_discount), created_at, updated_at

## 5. pledges

- id, project_id, user_id, amount, reward_id (nullable), stripe_payment_id, status, created_at, updated_at

## 6. comments

- id, project_id, user_id, parent_id (nullable, 대댓글), body, created_at, updated_at

## 7. updates

- id, project_id, user_id, title, body, created_at, updated_at

---

## 보완: 제약·인덱스 권장

| 테이블 | PK | FK | 인덱스 권장 |
|--------|----|----|-------------|
| users | id (UUID) | - | email UNIQUE |
| projects | id (UUID) | user_id → users | user_id, status, category, created_at |
| fundings | id (UUID) | project_id → projects | project_id UNIQUE |
| rewards | id (UUID) | project_id → projects | project_id |
| pledges | id (UUID) | project_id, user_id | project_id, user_id, created_at |
| comments | id (UUID) | project_id, user_id, parent_id | project_id, created_at |
| updates | id (UUID) | project_id, user_id | project_id, created_at |

- **민감 정보**: 카드 번호·CVC 등 저장 금지. 추후 결제 도입 시 결제 대행사 payment_id만 저장.
- **소프트 삭제**: projects 등 필요 시 `deleted_at` 컬럼 추가 검토.
