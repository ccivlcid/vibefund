# VibeFund 스키마 개요

PRD v2.0 기능 요구사항을 반영한 테이블·컬럼 요약입니다.  
**운영**: Supabase PostgreSQL. 실제 DDL은 Supabase SQL 에디터 또는 마이그레이션으로 관리합니다.  
**결제 미적용**: pledges 테이블은 Phase 2 또는 추후 결제 도입 시 사용.

## 1. users

- id, email, name, avatar_url, provider (oauth), **role** (user / admin — 관리자 페이지 접근용), created_at, updated_at

## 2. projects

- id, user_id, title, short_description, description, service_url, **category** (AI/SaaS/Automation/No-Code), thumbnail_url, **status** (Prototype/Beta/Live), **approval_status** (pending/approved/rejected — 관리자 검수용), created_at, updated_at, deleted_at (선택)

## 3. fundings

- id, project_id, goal_amount, deadline, min_pledge_amount, created_at, updated_at

## 4. rewards

- id, project_id, name, description, amount, type (beta/lifetime/subscription_discount), created_at, updated_at

## 5. pledges

- id, project_id, user_id, amount, reward_id (nullable), stripe_payment_id, status, created_at, updated_at

## 6. comments

- id, project_id, user_id, parent_id (nullable, 대댓글), body, created_at, updated_at

## 7. updates

- id, project_id, user_id, title, body, created_at, updated_at (Dev Log)

## 8. verification_responses (PRD v2.0 — 검증 질문 응답)

- id, project_id, user_id, **question_key** (q1_use_intent / q2_monthly_pay / q3_improvement 등), **answer** (텍스트 또는 코드화된 값), created_at  
- 고정 3문항: (1) 사용 의향 (2) 월 지불 가능 금액 (3) 가장 개선이 필요한 부분. Vibe Score 계산·집계에 사용.

## 9. comment_reports (댓글 신고)

- id, comment_id, reporter_user_id, reason (선택), status (pending/resolved_dismissed/resolved_deleted), created_at, updated_at  
- F-054 관리자 신고 처리용.

## 10. vibe_scores (PRD v2.0 — Vibe Score 1.0)

- id, project_id, **score** (숫자), **calculated_at**, 메타(선택)  
- 또는 매 요청 시 뷰/집계 쿼리로 계산. Phase 1에서는 단순 가중치(체험 수·검증 응답·댓글 등) 반영.

**랭킹**: 별도 테이블 없이 projects + vibe_scores + comments/updates 집계로 랭킹 쿼리 또는 materialized view 구성 가능.

## 보완: 제약·인덱스 권장

| 테이블 | PK | FK | 인덱스 권장 |
|--------|----|----|-------------|
| users | id (UUID) | - | email UNIQUE |
| projects | id (UUID) | user_id → users | user_id, status, category, created_at |
| fundings | id (UUID) | project_id → projects | project_id UNIQUE |
| rewards | id (UUID) | project_id → projects | project_id |
| pledges | id (UUID) | project_id, user_id | project_id, user_id, created_at |
| comments | id (UUID) | project_id, user_id, parent_id | project_id, created_at |
| updates | id (UUID) | project_id, user_id | project_id, created_at |
| verification_responses | id (UUID) | project_id, user_id | project_id, question_key, created_at |
| comment_reports | id (UUID) | comment_id, reporter_user_id | comment_id, status |
| vibe_scores | id (UUID) | project_id | project_id, calculated_at |

- **민감 정보**: 카드 번호·CVC 등 저장 금지. 추후 결제 도입 시 결제 대행사 payment_id만 저장.
- **소프트 삭제**: projects 등 필요 시 `deleted_at` 컬럼 추가 검토.
