# VibeFund 운영 메뉴얼 개요

## 1. 운영 범위 (MVP)

- **관리자 페이지**: 관리자 전용 로그인·권한(role)으로 접근. 일반 유저는 접근 불가.
- **대시보드**: 회원 수, 프로젝트 수, 검수 대기 건 등 KPI 요약
- **회원 관리**: 회원 목록·검색·상세, 정지·권한 변경 등
- **프로젝트 검수**: 신규·수정 프로젝트 목록, 승인/반려/비공개 처리, 반려 사유 입력
- **댓글·신고 관리**: 신고 접수 댓글 조회·삭제/유지 처리
- (현재) 결제·정산 없음 — 추후 결제 도입 시 정산·환불은 관리자 페이지에서 처리
- 고객 문의·신고 처리

## 2. 수수료·정산 (PRD §8)

- 펀딩 성공 시 플랫폼 수수료 5%
- 프리미엄 노출·Pro 계정·구독 수수료 정책 적용 시 운영 절차 정의 예정

## 3. 문서 보강 예정

- 정산 주기·방법
- 환불·취소 정책
- 프로젝트 삭제·비공개 처리
- 장애·긴급 연락 체계

## 4. Vercel·배포 운영

- **배포 트리거**: Git 푸시 시 Vercel이 빌드·배포 실행.
- **운영 절차**: 프로덕션 배포 전 Vercel 대시보드에서 빌드 결과 확인. 실패 시 빌드 로그 확인 후 수정·재푸시 또는 롤백.
- **환경·시크릿**: Vercel 프로젝트 설정에서 환경 변수(Production/Preview) 정기 점검. 퇴사자·권한 변경 시 접근 권한 갱신.
- **롤백**: Vercel 대시보드 → Deployments에서 이전 성공 배포로 롤백.

## 5. 보완: 운영 체크리스트

| 구분 | 항목 |
|------|------|
| 일일 | Supabase·Vercel 에러 로그 확인, 빌드 실패 건 확인 |
| 주간 | KPI 집계(프로젝트 수 등). 결제 도입 후 펀딩 성공 건수·누적 금액 추가 |
| 배포 시 | Vercel 빌드 로그·배포 결과 확인, 필요 시 Preview 검증 후 Production |
| 프로젝트 검수 | 관리자 페이지에서 신규 프로젝트 URL·콘텐츠 검토, 승인/반려/비공개 |
| 장애 시 | 상태 페이지·공지, 고객 문의 채널 안내. 배포 원인 시 Vercel 로그·롤백 검토 |
