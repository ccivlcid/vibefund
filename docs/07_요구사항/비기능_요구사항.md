# VibeFund 비기능 요구사항 (PRD §7 반영)

## 1. 성능

| 항목 | 요구사항 |
|------|----------|
| 페이지 로딩 | 3초 이내 |
| 반응형 | 모바일·태블릿·데스크톱 지원 |

## 2. 보안

| 항목 | 요구사항 |
|------|----------|
| 결제 | 현재 미적용. 추후 도입 시 PCI 준수(결제 대행사 위임) |
| 인증 | 백엔드 인증 API로 이메일·OAuth(Google) 처리, JWT/세션·비밀번호 해시 등 안전한 처리 |

## 3. 가용성

| 항목 | 요구사항 |
|------|----------|
| 서비스 가동률 | 99% 이상 목표 |

## 4. CI/CD (Vercel)

| 항목 | 요구사항 |
|------|----------|
| 배포 | Vercel Git 연동으로 푸시 시 빌드·배포 자동화 |
| 환경 | Vercel에서 Production/Preview 분리, 환경별 환경 변수 관리 |
| 배포 타깃 | Next.js → Vercel 배포 |

## 5. 추후 정의 예정

- 로그·모니터링
- 백업·복구
- 다국어·접근성

## 6. 보완: 상세 목표

| 항목 | 목표 | 측정 방법 |
|------|------|-----------|
| 페이지 로딩 | 3초 이내 | LCP 등 Core Web Vitals |
| 반응형 | 320px~1920px 대응 | 뷰포트별 QA |
| 가용성 99% | 월간 다운타임 약 7시간 이내 | 업타임 모니터링 |
| 보안 | 카드 데이터 미보관, HTTPS 전 구간 | Stripe·인프라 설정 검토 |
