# VibeFund 기능 요구사항 (PRD v2.0 §6 반영)

**API 엔드포인트·인증·에러 형식**: [02_기술정보/API_명세_개요.md](../02_기술정보/API_명세_개요.md) 참고.

## 6.1 사용자 관리

- **인증**: Supabase Auth 미사용. **백엔드 인증 API**로 구현 (회원가입/로그인/로그아웃·JWT 또는 세션 쿠키).

| ID | 요구사항 | 상세 |
|----|----------|------|
| F-001 | 회원가입 | 이메일 기반 가입, OAuth(Google) 지원 — 백엔드 API |
| F-002 | 로그인/로그아웃 | 백엔드 인증 API, JWT 또는 세션 기반 |
| F-003 | 프로필 관리 | 이름, 아바타 등 프로필 수정 |

## 6.4 Vibe Score 1.0 (Phase 1)

- 단순 가중치 기반 점수.
- **계산 요소(예)**: 체험 수, 검증 질문 응답, 댓글 수·반응 등. 상세 공식은 별도 기술 문서.

### 인수 조건 예시 (AC)

- **F-001**: 이메일 중복 가입 불가, OAuth 시 기존 계정 연동 가능
- **F-010**: 서비스 URL 필수, 카테고리(AI/SaaS/Automation/No-Code), 상태(Prototype/Beta/Live) 선택
- **F-011**: 검증 질문 3문항 응답 저장·집계, 대댓글 parent_id 연동
- **F-030**: 신고 시 관리자 목록에 노출, 삭제/유지 처리
- **F-040**: 로그인 사용자만 접근, 본인 데이터만 노출
- **F-050**: role=admin 구분, 미권한 시 403 또는 리다이렉트
