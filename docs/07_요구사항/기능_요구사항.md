# VibeFund 기능 요구사항 (PRD §6 반영)

**API 엔드포인트·인증·에러 형식**: [02_기술정보/API_명세_개요.md](../02_기술정보/API_명세_개요.md) 참고.

## 6.1 사용자 관리

- **인증**: Supabase Auth 미사용. **백엔드 인증 API**로 구현 (회원가입/로그인/로그아웃·JWT 또는 세션 쿠키).

| ID | 요구사항 | 상세 |
|----|----------|------|
| F-001 | 회원가입 | 이메일 기반 가입, OAuth(Google) 지원 — 백엔드 API |
| F-002 | 로그인/로그아웃 | 백엔드 인증 API, JWT 또는 세션 기반 |
| F-003 | 프로필 관리 | 이름, 아바타 등 프로필 수정 |

## 6.2 프로젝트 등록

| ID | 요구사항 | 상세 |
|----|----------|------|
| F-010 | 프로젝트 생성 | 제목, 한 줄 설명(필수), 상세 설명, 서비스 URL(필수), 카테고리, 썸네일 |
| F-011 | 미리보기 자동 생성 | OpenGraph 썸네일, iframe 미리보기, 모바일 뷰 |
| F-012 | 프로젝트 상태 | Prototype / Beta / Live |

## 6.3 펀딩 기능

| ID | 요구사항 | 상세 |
|----|----------|------|
| F-020 | 펀딩 목표 설정 | 목표 금액, 마감 기한, 최소 후원 금액 |
| F-021 | 리워드 설정 | 베타 접근권, 평생 이용권, 구독 할인권 등 |
| F-022 | 결제 처리 | **현재 범위 제외** — 추후 Stripe 등 도입 예정 |
| F-023 | 펀딩 상태 표시 | 목표·마감·리워드 안내, 진행률(현재는 0% 또는 표시만) |

## 6.4 커뮤니티 기능

| ID | 요구사항 | 상세 |
|----|----------|------|
| F-030 | 댓글 | 사용자 댓글, 제작자 답변 |
| F-031 | 프로젝트 업데이트 | 제작자 게시 |

## 6.5 유저 페이지 (마이페이지)

| ID | 요구사항 | 상세 |
|----|----------|------|
| F-040 | 유저 페이지 진입 | 로그인 사용자 전용. 본인만 접근 가능 |
| F-041 | 프로필 보기·수정 | 이름, 아바타, 소개 등 (F-003과 연동) |
| F-042 | 내 프로젝트 목록 | 본인이 등록한 프로젝트 목록·상태·편집 진입 |
| F-043 | 후원/관심 목록 | (결제 미도입 시) 관심 프로젝트 등. 결제 도입 후 내 후원 목록 |
| F-044 | 내 댓글·활동 | 본인이 쓴 댓글 목록 또는 알림 요약 |

## 6.6 관리자 페이지

| ID | 요구사항 | 상세 |
|----|----------|------|
| F-050 | 관리자 인증 | 관리자 전용 로그인 또는 role 구분(admin). 일반 유저는 접근 불가 |
| F-051 | 대시보드 | KPI 요약: 회원 수, 프로젝트 수, 신규/승인 대기 등 |
| F-052 | 회원 관리 | 회원 목록·검색·상세(필요 시). 정지·권한 변경 등 |
| F-053 | 프로젝트 검수 | 프로젝트 목록·검색, 승인/반려/비공개 처리, 반려 사유 입력 |
| F-054 | 댓글·신고 관리 | 댓글 목록, 신고 접수 건 처리(삭제·유지 등) |
| F-055 | (추후) 정산·결제 | 결제 도입 시 정산 현황·환불 처리 등 |

## 보완: 우선순위·인수 조건 예시

| 우선순위 | 범위 | 비고 |
|----------|------|------|
| P0 (MVP 필수) | F-001~F-003, F-010~F-012, F-020~F-021, F-023, F-030, F-040~F-042 | 로그인·프로젝트·펀딩·댓글·유저 페이지 |
| P1 | F-031, F-043~F-044, F-050~F-054 | 업데이트·유저 활동·관리자 페이지(검수·회원·댓글 관리) |
| 추후 | F-022, F-055 | 결제·정산 |

### 인수 조건 예시 (AC)

- **F-001**: 이메일 중복 가입 불가, OAuth 시 기존 계정 연동 가능
- **F-010**: 서비스 URL 필수, 유효 URL 형식 검증
- **F-022**: (추후) 결제 완료 시 Pledge 레코드 생성, 진행률 자동 반영
- **F-030**: 비로그인 댓글 허용 여부는 정책으로 정의
- **F-040**: 유저 페이지는 로그인 사용자만 접근, 본인 데이터만 노출
- **F-050**: 관리자 페이지는 role=admin 등으로 구분, 미권한 시 403 또는 리다이렉트
